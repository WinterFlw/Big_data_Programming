## 현재 문제 진단

```
현재 파이프라인의 한계:

1) SHAP이 서브워드 조각을 Top-5로 뽑음
   "shithole" → ["sh", "ith", "ole"]  ← 의미 없는 파편
   Top-5의 27~38%가 2글자 이하 서브워드

2) 모델이 표면 비하어에 편중
   overlap=1.0 케이스: 전부 human rationale 1~3개 (욕설 1개만 있는 쉬운 문장)
   overlap<0.3 케이스: human rationale 19~34개 (문장 전체가 맥락적 혐오)

3) LIME은 단어 단위지만, 개별 단어의 독립 기여만 측정
   "those people should go back" → 단어를 하나씩 빼봄
   → 어떤 단어 하나를 빼도 혐오 예측이 크게 안 변함
   → 모든 단어의 기여도가 낮게 측정됨
   → 실제로는 "단어들의 조합"이 혐오를 만드는 건데 이걸 못 잡음
```

## 개선 전략 3단계

### 개선 1: 서브워드 → 단어 Aggregation (XAI 측정 보정)

**문제:** SHAP은 토크나이저 단위로 점수를 매기므로 `nig` + `ger` = 2개 토큰
**해법:** 서브워드 점수를 원래 단어로 합산(sum) 후 Top-5 추출

```python
# 현재 (토큰 단위)
tokens:  ["nig", "ger", "s", "are", "trash"]
scores:  [0.31,  0.28,  0.05, 0.12,  0.24]

# 개선 (단어 단위 aggregation)
words:   ["niggers",      "are",  "trash"]
scores:  [0.31+0.28+0.05, 0.12,   0.24]
       = [0.64,           0.12,   0.24]
```

**구현 위치:** `experiment_xai.py`
- `run_shap_explanations()` 내부에서 토큰→단어 매핑 후 점수 합산
- BERT는 `##` prefix, RoBERTa는 `Ġ` prefix로 서브워드를 구분
- LIME은 이미 `split_expression=r"\s+"` (공백 기준)이라 단어 단위 → 변경 불필요

**효과:** Human Rationale overlap 수치가 정확해짐 + Top-5가 해석 가능한 단어가 됨

### 개선 2: Comprehensiveness / Sufficiency 마스킹 검증

**아이디어 (질문에서 언급한 "마스킹으로 문맥 확인"):**
모델이 정말로 특정 토큰에 의존하는지, 아니면 맥락 전체를 보는지를
마스킹 실험으로 정량적으로 검증할 수 있음.

**두 가지 마스킹 메트릭 (DeYoung et al., 2020):**

```
Comprehensiveness (포괄성):
  "Top-5 토큰을 제거했을 때 예측 확률이 얼마나 떨어지는가?"

  원본 예측확률:     P(hate | "those dirty immigrants should leave")  = 0.85
  Top-5 제거 후:     P(hate | "[MASK] [MASK] [MASK] should leave")    = 0.40
  Comprehensiveness: 0.85 - 0.40 = 0.45

  → 높을수록 "Top-5가 진짜 중요한 토큰이었다" (XAI 설명이 충실)
  → 낮으면 "Top-5를 빼도 예측이 안 변함" (다른 곳에 의존하고 있었음)


Sufficiency (충분성):
  "Top-5 토큰만 남기고 나머지를 다 제거했을 때 예측이 유지되는가?"

  원본 예측확률:     P(hate | "those dirty immigrants should leave")  = 0.85
  Top-5만 유지:      P(hate | "dirty immigrants [MASK] leave")        = 0.72
  Sufficiency:       0.85 - 0.72 = 0.13

  → 낮을수록 "Top-5만으로 충분히 예측 가능" (집중된 설명)
  → 높으면 "Top-5만으로는 부족, 맥락이 더 필요함" (분산된 맥락 의존)
```

**추가 마스킹 실험: Slur-Free Prediction (표면 토큰 의존도 측정)**

```
"모든 비하어를 마스킹했을 때 모델이 여전히 맥락으로 혐오를 잡는가?"

원본:              "niggers are ruining this country"  → P(hate) = 0.92
비하어 마스킹:      "[MASK] are ruining this country"   → P(hate) = ?

Case A: P(hate) = 0.78  → 맥락도 어느 정도 캐치 (양호)
Case B: P(hate) = 0.35  → 비하어에 완전 의존 (맥락 실패)
```

**구현 위치:** `experiment_xai.py`에 `_compute_masking_metrics()` 함수 추가
- XAI Step 5와 Step 6 사이에 삽입
- 기존 SHAP/LIME 결과의 Top-5를 그대로 활용하므로 추가 모델 학습 불필요
- Baseline vs Improved 비교 가능 → "VADER가 맥락 의존도를 높였는가" 검증

### 개선 3: Rationale-aware Attention Loss (모델 학습 개선)

**핵심 아이디어:**
현재 모델은 `CrossEntropyLoss(예측, 정답)`으로만 학습함.
"정답을 맞히기만 하면 됨" → 표면 비하어만 봐도 정답을 맞출 수 있음 → 맥락 무시

Human rationale을 attention supervision으로 추가하면:
"정답도 맞히고, 인간이 중요하다고 본 토큰에 집중해야 함" → 맥락 유도

```python
# 현재
loss = CrossEntropyLoss(logits, labels)

# 개선
classification_loss = CrossEntropyLoss(logits, labels)
attention_loss = compute_rationale_loss(attention_weights, rationale_mask)
loss = classification_loss + α * attention_loss
```

**Rationale Loss 계산 방식:**

```
입력 문장:  "those dirty immigrants should all go back to their countries"
토큰화:     [CLS] those dirty immigrants should all go back to their countries [SEP]

Human Rationale Mask (3인 과반수):
             0     0     1          1        0   0  1    0  0     1         0
             ─────────────────────────────────────────────────────────────────
             ※ "dirty", "immigrants", "back", "countries"에 2명 이상 표시

BERT Attention (마지막 레이어, 헤드 평균):
  [CLS]의 attention 분포:
             0.05  0.03  0.25       0.12     0.02 0.01 0.08  0.02  0.03  0.15  0.01
             ─────────────────────────────────────────────────────────────────────────

Rationale Loss = BinaryCrossEntropy(
    attention_weights / max(attention_weights),  # 정규화
    rationale_mask                                # 0/1 타겟
)

→ attention이 rationale 토큰에 집중하도록 학습!
→ "dirty", "immigrants", "back", "countries"에 높은 attention을 줘야 loss가 줄어듦
```

**모델 forward 변경:**

```python
# 현재 HybridSentimentClassifier.forward()
def forward(self, input_ids, attention_mask, vader):
    outputs = self.encoder(input_ids=input_ids, attention_mask=attention_mask)
    pooled = outputs.last_hidden_state[:, 0, :]
    combined = torch.cat([pooled, vader], dim=1)
    ...
    return self.out(combined)  # logits만 반환

# 개선: attention도 함께 반환
def forward(self, input_ids, attention_mask, vader, return_attention=False):
    outputs = self.encoder(
        input_ids=input_ids,
        attention_mask=attention_mask,
        output_attentions=True  # ★ attention weight 추출
    )
    pooled = outputs.last_hidden_state[:, 0, :]
    combined = torch.cat([pooled, vader], dim=1)
    ...
    logits = self.out(combined)

    if return_attention:
        # 마지막 레이어의 attention, 전체 헤드 평균
        # shape: (batch, heads, seq_len, seq_len) → (batch, seq_len)
        last_attention = outputs.attentions[-1]          # (B, H, S, S)
        cls_attention = last_attention[:, :, 0, :]       # [CLS] → all tokens
        cls_attention = cls_attention.mean(dim=1)        # 헤드 평균 → (B, S)
        return logits, cls_attention
    return logits
```

**학습 루프 변경 (train_neural_model):**

```python
# 현재
for batch in train_loader:
    logits = _forward_batch(model, batch, device)
    loss = criterion(logits, labels)

# 개선
for batch in train_loader:
    if batch.get("rationale_mask") is not None:
        logits, attention = _forward_batch(model, batch, device, return_attention=True)
        classification_loss = criterion(logits, labels)
        rationale_loss = compute_rationale_loss(attention, batch["rationale_mask"])
        loss = classification_loss + config.rationale_loss_weight * rationale_loss
    else:
        # rationale이 없는 샘플(normal 등)은 기존 방식
        logits = _forward_batch(model, batch, device)
        loss = criterion(logits, labels)
```

**주의사항:**
- Normal 클래스는 rationale이 0건 → rationale loss 미적용 (classification만)
- Hatespeech/Offensive는 95% 커버 → 대부분 적용 가능
- α (rationale_loss_weight)는 0.1~0.5 범위에서 tuning
- 너무 크면 attention이 rationale에만 과적합 → 분류 성능 하락

### LIME의 맥락 한계와 대안

**왜 LIME은 맥락을 못 잡나?**

LIME의 perturbation 방식:
```
원본: "those dirty immigrants should all go back"

Perturbation 1: "those [삭제] immigrants should all go back"  → P(hate)=0.80
Perturbation 2: "those dirty [삭제] should all go back"       → P(hate)=0.72
Perturbation 3: "those dirty immigrants [삭제] all go back"   → P(hate)=0.83
...

LIME은 각 단어를 독립적으로 빼보고, 예측 변화를 선형 회귀로 근사합니다.
→ "dirty"를 빼도 0.85→0.80 (약간만 변화)
→ "immigrants"를 빼도 0.85→0.72 (좀 변화)

문제: "dirty" + "immigrants" + "go back"이 함께 있어야 혐오인 건데,
      LIME은 "개별 단어의 기여"만 측정하므로 조합 효과(interaction)를 놓침.
```

**LIME 자체를 고칠 수는 없나?**

LIME의 구조적 한계는 "additive feature attribution" (개별 피처의 독립 기여 가정)에 있음.
이걸 고치려면 LIME 자체를 바꿔야 하는데, 그보다는:

1. **마스킹 검증(개선 2)으로 이 한계를 정량적으로 보여주는 게** 논문에서 더 가치 있음
2. **SHAP의 Partition Explainer**는 계층적으로 토큰을 묶어서 설명하므로 LIME보다 나음
   (현재 이미 사용 중)
3. **Rationale-aware Training(개선 3)으로 모델 자체가 맥락을 보게 만들면**
   LIME으로 측정해도 맥락 토큰의 기여도가 자연스럽게 올라감

즉, LIME을 고치는 게 아니라:
- 마스킹으로 LIME의 한계를 "증명"하고 (학술적 기여)
- 모델 자체를 개선해서 결과적으로 LIME 설명도 좋아지게 만드는 전략

## 구현 순서 (의존성 기준)

```
Phase A: XAI 측정 보정 (모델 재학습 불필요)
  A-1. 서브워드 → 단어 Aggregation          [experiment_xai.py]
  A-2. Comprehensiveness/Sufficiency 마스킹  [experiment_xai.py]
  A-3. Slur-Free Prediction 실험             [experiment_xai.py]

  B-1. Rationale mask를 DataFrame에 추가     [experiment_core.py - load_processed_dataframe]
  B-2. Dataset 클래스에 rationale 전달       [experiment_core.py - TransformerTextDataset/HybridTextDataset]

Phase C: 모델 학습 개선
  C-1. forward()에 output_attentions 추가    [experiment_core.py - 3개 모델 클래스]
  C-2. compute_rationale_loss() 구현         [utils.py]
  C-3. train_neural_model() 수정             [experiment_core.py]
  C-4. ExperimentConfig에 rationale_loss_weight 추가
  C-5. 전체 재학습 + 벤치마크

Phase D: 재검증
  D-1. XAI 재실행 (Phase A 개선된 측정 기준)
  D-2. Before(v2) vs After(v3) 비교
```
