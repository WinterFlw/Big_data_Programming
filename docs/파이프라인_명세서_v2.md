## 0. 한 줄 요약 (척추 메시지)

본 프로젝트는 *"단어 단서뿐 아니라 맥락 단서까지 함께 학습한 혐오표현 탐지 모델"*이 베이스 대비 분류 성능과 판단 투명성 모두에서 향상됨을, Rationale-aware Attention Loss와 VADER 감성 피처를 BERT/RoBERTa fine-tuning에 결합하고, **인간 라벨 의존을 최소화한 자동 XAI 4축 평가 체계**로 정량 입증한다.

베이스 BERT가 단어 단서(특히 비하어)에 과의존하는 진단(H1)에서 출발하여, 두 개선 기법(Attention Loss + VADER)이 단어 신호는 유지하면서 맥락 단서를 추가로 학습시키는지를 8조건 풀 ablation과 자동 XAI 4축으로 검증한다.

## 1. (3) 초안 대비 핵심 변경사항

| # | 변경 항목 | 변경 내용 | 변경 사유 |
|---|---------|---------|---------|
| 1 | **Slur-Masking augmentation** | 학습 단계에서 제거 | [MASK] shortcut 위험·하드웨어·학습 시간 부담 |
| 2 | **Ablation 구조** | 4조건 → 8조건 (BERT × 4 + RoBERTa × 4) | 사전학습 모델 강건성 검증 |
| 3 | **Stratified split** | label 단일 stratified | source는 모델 입력에 안 들어가므로 직접 leakage 차단됨, source-별 분리 보고로 보강 |
| 4 | **XAI 4축 재정의** | Attribution / Faithfulness / Context Learning(자동) / Plausibility(보조) | 인간 정의 카테고리 의존 최소화 |
| 5 | **자동 메트릭 신규** | CI / IS / MSS 정식 채택 | 모델 내부 토큰 동역학으로 맥락 학습 정량화 |
| 6 | **α 그리드 조건** | B_B 조건에서 최적값 결정 후 다른 조건 동일 적용 | (3) 초안의 A_B 명시는 정의상 모순 (A는 Attn 미사용) |
| 7 | **VADER 표현** | "Cheng (2022) 선행연구 기반 사전 가설"로 일관 표기 | "XAI 진단 결과" 표현 금지 |
| 8 | **Confidence weighting** | 사용 안 함 | 본 데이터셋 평균 agreement 0.814로 라벨 노이즈 적어 효과 미미 예상 |
| 9 | **L_target multi-task** | D_B 부가 실험으로 제한 운영 | 메인 8조건 ablation의 단순성 유지 |

## 2. 핵심 원칙 — 모델 입력 단일 소스

### 원칙
모델은 오직 텍스트(post_tokens)를 입력으로 받는다. 학습 시점에는 label, rationale, target이 손실 함수의 supervision 신호로 사용되지만 모델 입력에는 포함되지 않는다. 추론 시점에는 텍스트만 주어져도 모델이 동작한다.

### 학습 시 데이터 흐름

```
┌─────────────────────────────────────────────────────────────┐
│ Raw Sample                                                  │
│  - post_tokens (텍스트)        ── 입력으로 들어감             │
│  - label                       ── 메인 정답                   │
│  - rationale (3인 마스크)      ── Attn Loss supervision       │
│  - target (multi-label)        ── Aux Loss supervision (부가) │
│  - source                      ── 미사용                      │
│  - agreement                   ── 미사용 (EDA 분석만)         │
│  - annotators                  ── 미사용                      │
└─────────────────────────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│ Input Pipeline (텍스트만)                                    │
│   post_tokens ──┬──→ Tokenizer → BERT/RoBERTa → [CLS] 768d   │
│                 │                                            │
│                 └──→ VADER SentimentIntensityAnalyzer        │
│                          └─→ {pos, neg, neu, compound} 4d    │
└─────────────────────────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│ Concat & Encode                                             │
│   [CLS 768] ⊕ [VADER 4]  →  772d  (C/D 조건만)               │
│   → Dropout(0.1) → Linear(772→256) → ReLU                    │
└─────────────────────────────────────────────────────────────┘
                  │
            ┌─────┴─────┐
            ▼           ▼
┌─────────────────┐  ┌──────────────────────────┐
│ Main Head       │  │ Aux Head (D_B 부가만)    │
│ Linear(256, 3)  │  │ Linear(256, T_target)    │
│ → 3-class       │  │ → target multi-label     │
└─────────────────┘  └──────────────────────────┘
            │           │
            ▼           ▼
        L_cls       L_target  +  L_attn (CLS attention vs rationale)

   L_total = L_cls + α·L_attn + β·L_target
```

### 추론 시 데이터 흐름


## 5. STAGE 2 — 전처리

### 5.1 텍스트 정제
- URL → `<url>`, 멘션 → `<user>`, 숫자 → `<number>`
- HTML 엔티티 디코딩, 특수문자 제거, 소문자 변환

### 5.2 토큰화
- BERT/RoBERTa: WordPiece / BPE
- TF-IDF 베이스라인: 별도 (소문자 + 불용어 + 어간)

- 라벨 통합: 3인 다수결 ≥2, 동률 제외

- 원문 토큰의 rationale을 모든 서브워드에 복제

## 7. STAGE 4 — 학습 (핵심)

### 7.1 데이터 증강 정책
**학습 데이터 증강 없음.** 원본 19,192건만 사용.

(3) 초안의 Slur-Masking augmentation은 [MASK] shortcut 위험·하드웨어 제약·학습 시간 부담으로 학습에서 제외한다. 비하어 마스킹은 평가 단계의 정성 분석(Slur-Free Prediction 사례)에만 활용한다.

### 7.2 8조건 ablation 매트릭스

**BERT 패밀리**
| | VADER X | VADER O |
|--|--------|--------|
| Attn X | A_B: BERT + MLP | C_B: BERT + VADER + MLP |
| Attn O | B_B: A_B + Attn Loss | D_B: A_B + VADER + Attn |

**RoBERTa 패밀리** (동일 구조)
| | VADER X | VADER O |
|--|--------|--------|
| Attn X | A_R | C_R |
| Attn O | B_R | D_R |

### 7.3 모델 구조 명세

**A_B / B_B / A_R / B_R (768d 입력)**
```
post_tokens → Tokenizer → BERT/RoBERTa → [CLS] 768d
  → Dropout(0.1) → Linear(768, 256) → ReLU → Linear(256, 3) → Main Head
```

**C_B / D_B / C_R / D_R (772d 입력)**
```
post_tokens ─┬─→ Tokenizer → BERT/RoBERTa → [CLS] 768d ─┐
             │                                          ├─→ ⊕ → 772d
             └─→ VADER → 4d ───────────────────────────┘
  → Dropout(0.1) → Linear(772, 256) → ReLU → Linear(256, 3) → Main Head
```

**듀얼 헤드 (D_B 부가 실험만)**
```
... → Linear(772 또는 768, 256) → ReLU
                    │
        ┌───────────┴───────────┐
        ▼                       ▼
  Main Head                  Aux Head
  Linear(256, 3)             Linear(256, T_target)
  → 3-class                  → target multi-label
```

모든 조건에서 분류 헤드 구조 통일. VADER 유무는 입력 차원만 차이.

### 7.4 학습 손실

```
L_total = L_cls + α·L_attn + β·L_target
```

| 항 | 정의 | 사용 조건 |
|----|----|--------|
| L_cls | CrossEntropy(main_logits, label), class_weight balanced | 모든 조건 |
| α·L_attn | BCE(CLS_attention 분포, rationale_mask) — 토큰 단위 | B / D 조건 |
| β·L_target | BCE(aux_logits, target_multilabel) | D_B 부가 실험만 |

- α ∈ {0.0, 0.1, 0.3, 0.5, 0.7, 1.0} 그리드 — **B_B 조건에서 val Macro F1 기준 최적값 결정** 후 D_B / B_R / D_R 조건에 동일 적용
- β ∈ {0.0, 0.1, 0.3} (부가 실험)

### 7.5 Hyperparameter (변수 통제)

| 항목 | 값 | 비고 |
|------|----|----|
| Optimizer | AdamW | weight decay 0.01 |
| LR | 2e-5 | 튜닝 1e-5~5e-5 |
| Batch | 16 | M3 Max MPS |
| Epochs | 3~5 | early stop on val Macro F1 |
| Warmup ratio | 0.1 | linear |
| Dropout | 0.1 | |
| Max seq len | 128 | |
| Seed | 42, 123, 7 | 3회 반복 |
| Class weight | balanced | sklearn |

### 7.6 학습 방식

- **메인**: 전이학습 기반 full fine-tuning (12 layer + 분류 헤드 동시 업데이트)
- **통제 (Freeze Study)**: BERT 동결, 분류 헤드만 학습 → fine-tuning 기여도 분리

### 7.7 VADER 도입 근거 (명시)

VADER 결합은 Cheng (2022, Virginia Tech) 선행연구 기반 사전 가설이다. XAI 진단 결과로 도출된 피처 설계가 아니라, 선행 연구가 보고한 감성 정보 결합 효과를 본 연구가 ablation으로 통제 검증하는 구조다.

VADER는 텍스트로부터 추론 시점에 자동 계산되는 파생 피처이므로 "모델 입력 단일 소스" 원칙을 위반하지 않는다.

### 7.8 Annotator Agreement 활용 정책

agreement는 모델 학습에 사용하지 않는다. EDA 단계(p11)와 결과 분석 단계에서 데이터 특성(offensive 클래스의 평균 0.752로 가장 모호)을 보고하는 데에 한정한다. Confidence weighting은 본 데이터셋의 평균 agreement(0.814)가 충분히 높아 효과가 제한적일 것으로 판단해 학습 처치에서 제외하며, 노이즈가 큰 데이터셋에서의 검증은 향후 연구로 남긴다.

### 7.9 학습 분량

- 8조건 × 3 seed = 24회 학습
- Freeze Study 보조 실험
- D_B with/without L_target 부가 실험 (2조건 × 3 seed = 6회 학습)
- 추정 시간: 약 30~60시간 (M3 Max MPS)

## 8. STAGE 5 — XAI 4축 평가

### 8.1 축 1 — Attribution (어떤 토큰을 보는가)

#### SHAP (GradientExplainer)
- 토큰별 Shapley value $\phi_t$ 산출
- 샘플링: 클래스별 stratified 500 샘플 / 조건
- 인용: Lundberg & Lee (2017)

#### LIME
- 입력 섭동 기반 국소 선형 근사, 섭동 500회
- 인용: Ribeiro et al. (2016)

#### LOO (sanity check)
- 한 토큰씩 [MASK] → 예측 확률 변화량
- 짧은 문장(<30토큰) 200개 샘플링
- 인용: Li et al. (2016)

### 8.2 축 2 — Faithfulness (모델-attribution 충실성)

#### Comprehensiveness ↑
$$\text{Comp} = f(x)_y - f(x_{\setminus \text{top-k}})_y$$
- top-k 토큰 제거 시 예측 확률 변화량
- k = {5, 10, 20% of tokens} 다중 평가
- 클수록: 그 토큰이 진짜 결정에 중요

#### Sufficiency ↓
$$\text{Suff} = f(x)_y - f(x_{\text{top-k}})_y$$
- top-k 토큰만 남길 때 예측 변화
- 작을수록: top-k만으로 충분

#### MSS (Minimum Sufficient Subset Size)
$$\text{MSS} = \min_{S} |S| \quad \text{s.t.} \quad f(x_S)_y \geq \tau \cdot f(x)_y$$
- τ = 0.8
- 그리디 search 근사 (NP-hard 진짜 최소 보장 X)

**직관 비유**: 직소 퍼즐. 단순 그림은 1~2조각이면 알 수 있고, 복잡한 풍경화는 10조각이 필요하다. 단어 의존 모델은 MSS가 작고, 맥락 의존 모델은 크다.

인용: DeYoung et al. (2020) ERASER

### 8.3 축 3 — Context Learning (인간 의존 0, 본 연구 결정 카드)

본 프로젝트의 결정 카드. 인간 정의 카테고리(비하어 목록 등) 없이 모델 내부 동역학으로 맥락 학습 정량화.

#### CI (Concentration Index)
$$\text{CI} = \text{Gini}(|\phi_1|, |\phi_2|, \dots, |\phi_n|)$$
- 0(완전 균등) ~ 1(완전 집중)
- CI ↑ → 소수 토큰 집중 → 단어 의존
- CI ↓ → 분산 → 맥락 의존

**직관 비유**: 회의실에서 부장 한 명이 발언의 90%를 차지하면 CI 높음. 5명이 골고루 발언하면 CI 낮음. 결정 권한의 불평등 정도.

인용: Gini (1912) 분포 불평등 측정

#### IS (Interaction Strength)
$$\phi_{ij} = \mathbb{E}[f(x_{S \cup \{i,j\}}) - f(x_{S \cup \{i\}}) - f(x_{S \cup \{j\}}) + f(x_S)]$$
$$\text{IS} = \frac{1}{|P|} \sum_{(i,j) \in P} |\phi_{ij}|$$
- 샘플링: 50쌍 × 200 샘플 / 조건 (계산 비용 축소)
- IS ↑ → 토큰 시너지 강함 → 맥락 의존
- IS ↓ → 토큰 독립 → 단어 의존

**직관 비유**: 단거리 달리기는 한 명이 혼자 뛰지만, 축구는 11명의 패스 협업으로 골이 들어간다. 단어 의존 모델은 단거리 선수, 맥락 의존 모델은 축구 팀.

인용: Lundberg et al. (2020) Shapley interaction

#### Attention Rollout
$$A_{\text{rollout}} = \prod_{l=1}^{L} (0.5 A_l + 0.5 I)$$
- L = 12 layer
- 메트릭: rollout 분포 entropy (높을수록 분산)
- 거의 무료 (forward pass 1회)

인용: Abnar & Zuidema (2020) ACL

### 8.4 축 4 — Plausibility (보조 검증)

인간 rationale과의 부합도. 메인 결과 아님, HateXplain 강점 활용 차원.

- Token F1: 모델 top-k 토큰 vs human_mask
- IOU: $|M_{\text{model}} \cap M_{\text{human}}| / |M_{\text{model}} \cup M_{\text{human}}|$
- 임계값: top {10%, 20%, 30%} 다중 평가

인용: Mathew et al. (2021) HateXplain

### 8.5 보조 — Cross-method Consistency

**Overlap@5**: SHAP top-5 vs LIME top-5 일치율
- 두 attribution 방법의 안정성 확인 (단일 신뢰도 지표 아님)
- 임의 기준 60% (참고용)

## 10. 가설 (H1~H4)

- **H1 (진단)**: A_B의 SHAP 상위 토큰에 비하어 집중 + CI 높음 → 베이스 BERT는 단어 단서에 과의존하며 맥락 단서 활용이 부족
- **H2 (개선)**: D_B vs A_B에서 ΔF1 > 0, p < 0.05, RoBERTa(D_R vs A_R)에서도 일관 → 단어 + 맥락 결합 학습의 분류 성능 향상
- **H3 (입증)**: D_B의 자동 메트릭(CI ↓, IS ↑, MSS ↑)이 A_B 대비 통계 유의 + RoBERTa에서도 일관 → 단어 신호 유지 + 맥락 단서 추가 학습의 모델 내부 입증
- **H4 (보조)**: D_B + L_target multi-task 적용 시 hate↔offensive 분리 성능 향상 + Subgroup 균등성 ↑

## 부록 A — 평가위원 예상 질문 + 답변

**Q1**: BERT MLM도 마스킹과 유사한데 왜 별도 처치가 필요한가?
> MLM은 사전학습 단계의 일반 언어 표현 학습이고, fine-tuning 단계에서는 새로운 라벨 분포에 맞춰 가중치가 다시 움직이며 새로운 spurious correlation이 발생할 수 있다. 본 연구는 학습 단계의 마스킹 처치를 제거했지만, 평가 단계에서 비하어 마스킹 정성 사례 분석으로 모델의 비하어 의존도를 직접 측정한다.

**Q2**: 모델은 전이학습인가 fine-tuning인가?
> 전이학습 기반 full fine-tuning이다. 사전학습 가중치를 로드한 후 12개 Transformer layer 전체와 분류 헤드를 동시에 업데이트한다. 비교 통제군으로 BERT 가중치를 동결하고 분류 헤드만 학습하는 feature extraction 방식(Freeze Study)을 별도 운영하여 fine-tuning의 기여도를 정량 분리한다.

**Q3**: VADER는 메타 정보 아닌가?
> VADER는 텍스트로부터 추론 시점에 SentimentIntensityAnalyzer로 자동 계산되는 파생 피처다. 외부 메타 데이터가 아닌 텍스트의 함수이므로, 새로운 트윗이 주어졌을 때도 추가 정보 없이 동작한다.

**Q4**: target과 rationale은 모델 입력인가?
> 모두 모델 입력이 아니다. rationale은 Attention Loss의 supervision target, target은 부가 multi-task auxiliary loss의 supervision target으로 학습 시점에만 손실 함수에 들어간다. 추론 시에는 사용되지 않으며, 사용자는 텍스트만 제공하면 분류 결과를 얻는다.

**Q5**: source-stratified split을 왜 안 했나?
> source 컬럼은 모델 입력에 포함되지 않으므로 직접 leakage는 발생하지 않는다. 텍스트 스타일 차이에 의한 간접 leakage는 평가 단계에서 Twitter-only / Gab-only 성능을 분리 보고하여 검출한다.

**Q6**: agreement를 confidence weighting으로 학습에 반영하지 않은 이유?
> 본 데이터셋의 평균 agreement(0.814)가 충분히 높아 confidence weighting의 효과가 제한적일 것으로 판단했다. 노이즈가 큰 데이터셋에서의 검증은 향후 연구로 남긴다.

**Q7**: 자동 XAI 메트릭(CI/IS/MSS)의 학술 근거는?
> CI는 Gini(1912) 분포 불평등 측정, IS는 Lundberg(2020)의 Shapley interaction values, MSS는 DeYoung(2020) ERASER benchmark의 sufficiency 확장이다. 모두 학술적으로 검증된 표준 기법이다.
