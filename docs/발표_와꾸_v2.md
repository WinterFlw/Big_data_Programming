## 0. 척추 메시지 (모든 슬라이드 관통)

> *"단어 단서뿐 아니라 맥락 단서까지 함께 학습한 모델"*이 베이스 대비 분류 성능 + 판단 투명성 모두 향상됨을, 8조건 풀 ablation과 자동 XAI 4축으로 정량 입증한다.
>
> 베이스 BERT가 단어(특히 비하어)에 과의존하는 진단(H1)에서 출발하여, 두 개선 기법(Attention Loss + VADER)이 **단어 신호는 유지하면서 맥락 단서를 추가로 학습**시키는지를 검증한다.

## 2. 페이지별 와꾸 (p1~p26)

### p1. 프로젝트 수행 배경 + 필요성
- **헤드라인**: 비하어 매칭 모델은 맥락 혐오를 못 잡고, 규제 환경은 판단 투명성을 동시 요구한다
- 사회적 배경: 온라인 혐오표현 급증, 비하어 의존 모델의 맥락 오분류 (Davidson 2017)
- 규제: EU DSA(2024 시행), 한국 방심위 자동 필터링 권고
- 현장 문제: "왜 그렇게 판단했는가" 부재 → 모더레이션 검토·이의제기 대응 불가
- 시사점: 정확도 + 투명성 동시 만족 모델 필요 → 다음 슬라이드 목표

---

### p2. 프로젝트 목표
- **헤드라인**: 단어 + 맥락 결합 학습 모델이 베이스 대비 향상됨을 과학적 검증 프레임워크로 정량 입증
- 핵심 가설: 베이스 BERT는 단어에 *과의존*(H1) → 개선 모델은 단어 신호 유지 + 맥락 단서 *추가* 학습(H3)
- 두 개선 기법: Rationale-aware **Attention Loss** (학습 손실층) + **VADER concat** (모델 입력층)
- 통제 설계: **2 사전학습 모델 × 4조건 풀 ablation** (BERT × 4 + RoBERTa × 4 = 8조건)
- 검증 체계: **인간 라벨 의존을 최소화한 자동 XAI 4축** (Attribution / Faithfulness / Context Learning / Plausibility)
- 일반화: 본 평가 체계는 텍스트 분류 task 전반의 맥락 학습 검증에 적용 가능

---

### p3. 연구 질문 + 가설
- **헤드라인**: 4개 가설 — H1 진단 / H2 분류 향상 / H3 맥락 학습 입증 / H4 부가
- **RQ1/H1 (진단)**: A_B의 SHAP 상위에 비하어 집중 + CI 높음 → 단어 단서 과의존, 맥락 단서 활용 부족
- **RQ2/H2 (개선)**: D_B vs A_B에서 ΔF1 > 0, p < 0.05, RoBERTa 일관 → 단어 + 맥락 결합 분류 성능 향상
- **RQ3/H3 (입증)**: D_B의 자동 메트릭(**CI ↓, IS ↑, MSS ↑**) 통계 유의 + RoBERTa 일관 → 단어 신호 유지 + 맥락 추가 학습
- **RQ4/H4 (보조)**: D_B + L_target multi-task → hate↔offensive 분리 향상 + Subgroup 균등성 ↑
- 통과 기준: 자동 메트릭 다축 일관성 (단일 지표 의존 회피)

### p4. 관련 연구 ① — 혐오표현 탐지 흐름
- **헤드라인**: ML → DL → 사전학습 → XAI의 4단계 진화, 공통 한계는 단어 의존
- 1단계 ML: TF-IDF + LR/SVM (Davidson 2017) — 표면 어휘 한계
- 2단계 DL: CNN/BiLSTM + Word2Vec/GloVe — 단어 임베딩 문맥 민감성 부족
- 3단계 사전학습: BERT fine-tuning (Devlin 2019), RoBERTa (Liu 2019)
- 성능 비교: BERT 기반이 SVM/CNN/LSTM 대비 일관 우수
- **공통 한계**: 단어 의존 + 맥락 학습 강제 부재 → 본 연구의 진단 출발점

### p5. 관련 연구 ② — XAI + Rationale + Slur 의존

- → 본 연구는 이들의 학습 처치·자동 측정 결합

### p8. 전처리 파이프라인
- **헤드라인**: 정제 → 토큰화 → 라벨/Rationale 통합 → 서브워드 매핑 → Stratified split

### p12. 구현 모델 — 6+1 라인업
- **헤드라인**: 전통 ML 베이스 2 + DL 베이스 + 통제 + 제안 2 + 부가 = 6+1
- **베이스라인 ML**: TF-IDF + LR / TF-IDF + SVM (성능 하한선)
- **베이스라인 DL**: BERT-base + MLP (XAI 진단 기준점, A_B)
- **통제**: BERT + MLP — VADER ablation 통제 (분류 헤드 구조 통일)
- **제안**: BERT + VADER + MLP, RoBERTa + VADER + MLP
- **부가**: Freeze Study (BERT 동결, MLP만 학습) — fine-tuning 기여도 분리
- 모든 transformer 조건에서 분류 헤드 MLP 통일, VADER 유무는 입력 차원만 차이 (768 vs 772)

### p13. 모델 선정 이유 + 학습 방법 ⭐ 평가 핵심
- **헤드라인**: 전이학습 기반 **full fine-tuning** + 모델 입력은 텍스트만 (메타는 supervision으로만)
- BERT 선정 근거: 혐오 탐지 분야 우수 성능 (Devlin 2019), SHAP 호환, EDA에서 텍스트 임베딩 필요성 입증
- VADER 선정: **Cheng (2022, Virginia Tech) 선행연구 기반 사전 가설** *(XAI 진단 결과 X)*
- **학습 방식**: bert-base-uncased 사전학습 가중치 로드 → [CLS] 위에 분류 헤드 부착 → **12 Transformer layer 전체 + 분류 헤드 동시 업데이트**
- **통제 (Freeze Study)**: BERT 가중치 동결 + 분류 헤드만 학습 (feature extraction)
- **모델 입력 단일 소스 원칙**: 모델은 텍스트만 입력, rationale/target/source/agreement는 학습 supervision 또는 분석으로만 사용 → 추론 시 텍스트만 주어져도 동작
- VADER 4d는 텍스트로부터 자동 추출되는 파생 피처 → 본 원칙 부합

### p15. 모델 구현 + Freeze Study
- **헤드라인**: 6모듈 구조 유지, 듀얼 헤드 + Freeze 통제로 fine-tuning 기여도 분리
- 6모듈: experiment_core / eda / xai / dashboard + utils + run_experiments
- 듀얼 헤드 (D_B 부가만): Main Head (3-class) + Aux Head (target multi-label) — 추론 시 Main만 활성화
- L_attn = BCE(CLS attention 분포, rationale_mask) — 토큰 단위
- 재현성: torch/numpy/random seed 고정, 학습 로그 기록
- MPS 제약: SHAP은 CPU only 강제 (MPS 비호환)
- **Freeze Study**: BERT freeze + MLP만 학습 → 사전학습 표현 그대로 사용 vs 재학습 기여도 정량 분리
- 학습에 사용 안 함: Slur-Masking augmentation 제거, source 입력 X, agreement 미사용

### p16. 실험 설계 — 8조건 풀 ablation
- **헤드라인**: 2×2 (Attn × VADER) × 2 (BERT/RoBERTa) = 8조건 + 부가 실험
- BERT 패밀리:

  | | VADER X | VADER O |
  |--|--------|--------|
  | Attn X | A_B | C_B |
  | Attn O | B_B | **D_B** |

- RoBERTa 패밀리: A_R / B_R / C_R / D_R (동일 구조)
- 변수 통제: 시드/분할/하이퍼파라미터 동일 → 단일 변수만 변경
- 3회 반복 평균 ± std → 24회 학습 + Freeze + L_target 부가 = 약 30~60시간
- 통계 검정: Two-way ANOVA(BERT) + 3-way ANOVA(BERT/RoBERTa × Attn × VADER) + paired t-test (p<0.05)
- 두 기법 직교성: VADER = 모델 구조 / Attn = 학습 손실 → 다른 층위 개입, 상호작용 효과로 시너지/상쇄 측정

### p17. 베이스라인 모델 정의
- **헤드라인**: 두 수준의 베이스라인 — 전통 ML(하한선) + DL(XAI 진단 기준점)
- **수준 1 — 전통 ML**: TF-IDF + LR/SVM (scikit-learn) — 성능 하한선, 사전학습 효과 분리용
- **수준 2 — DL**: BERT-base + MLP (A_B) — XAI 진단 기준점, 개선 실험의 기준
- 학습 방식 차이: ML은 처음부터, DL은 **전이학습 기반 full fine-tuning**
- 동일 데이터·시드·split으로 공정 비교
- 1차 파이프라인 결과(2026-04-11): Macro F1 0.6822 (baseline 보유)

### p18. 성능 평가 지표 — 분류 + XAI 4축
- **헤드라인**: 분류 메트릭 + XAI 4축으로 다각 평가
- **분류**: Macro F1 (주, 클래스 불균형 공정), Precision/Recall, Confusion Matrix, Subgroup F1 (target/source 별)
- **XAI 축 1 — Attribution**: SHAP (GradientExplainer, 500 stratified/조건), LIME (섭동 500회), LOO (sanity check)
- **XAI 축 2 — Faithfulness**: Comprehensiveness↑, Sufficiency↓, **MSS** (그리디 search, τ=0.8) — DeYoung 2020 ERASER
- **XAI 축 3 — Context Learning ⭐**: **CI** (Gini 분포 집중도) / **IS** (Shapley interaction 50쌍×200, Lundberg 2020) / **Attn Rollout** (entropy, Abnar 2020) — 인간 의존 0
- **XAI 축 4 — Plausibility (보조)**: Token F1, IOU vs human rationale (Mathew 2021)
- 보조: Overlap@5 (SHAP-LIME 안정성), 정성: Slur-Free Prediction 사례 5~10건 (illustration)

### p25. 진행 중 발생한 문제점
- **헤드라인**: 4가지 도전 + 솔직한 (3) 초안 대비 조정 사항 명시
- **(3) 초안 대비 조정**: Slur-Masking augmentation 학습 제외 ([MASK] shortcut 위험 + 하드웨어), Stratified label 단일 (source 입력 X로 충분), Confidence weighting 미사용 (agreement 0.814로 효과 미미)
- Source 분포 차이(Gab 49% vs Twitter 8.7%) → 직접 leakage 입력 제외로 차단, 간접은 source-별 F1 분리
- SHAP MPS 비호환 → CPU only 강제, 클래스별 stratified 500 샘플로 비용 축소
- Offensive 클래스 모호성 (agreement 0.752) → 학습 처치 X, 결과 해석 시 어려움 인정
- IS (Shapley interaction) 계산 부담 → 50쌍 × 200 샘플로 축소
- L_target multi-task negative transfer 위험 → 부가 실험으로 분리, 결과 안 좋으면 한계로 솔직히 보고

### p26. 모델 개선 방향 + 향후 연구
- **헤드라인**: 단기는 그리드 확장·Focal Loss, 중장기는 Adversarial·한국어·LLM
- 단기 (학기 내):
  - α 그리드 추가 탐색 (B_B 조건 외 D_B에서도 별도 탐색 검토)
  - Focal Loss 도입 검토 (offensive 클래스 모호성 보완)
  - L_target β 그리드 세밀화
- 중장기 (학기 후):
  - Adversarial domain-invariant loss (gradient reversal로 source spurious 차단)
  - Confidence weighting을 노이즈 큰 데이터셋에서 검증
  - 한국어 데이터셋(K-HATERS, KOLD) 일반화 실험
  - LLM Chain-of-Thought 비교
  - Human-in-the-loop 모더레이션 시스템 통합
