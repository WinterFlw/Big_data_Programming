# 버전 A: 학술 논문 톤 (정제된 논증 구조)

## 1. 프로젝트 개요

### 1.1 수행 배경 및 필요성

스마트폰과 소셜미디어의 확산으로 온라인 공간은 일상적인 소통 채널로 자리 잡았다. 그러나 그 이면에서는 특정 집단을 겨냥한 혐오표현이 빠르게 확산되고 있다. 국내에서 혐오표현이 사회 문제로 본격 부상한 것은 2010년 전후로, 반다문화 성향의 인터넷 커뮤니티를 중심으로 외국인과 이주노동자를 향한 적대적 언어가 새로운 갈등 요인으로 떠올랐다. 이후 대상은 성별, 세대, 지역 등으로 확대되었고, 방송통신심의위원회에 접수되는 시정 요구 역시 꾸준히 증가하고 있다. 한 연구에서는 조사에 참여한 대학생 전원이 온라인에서 혐오표현을 접한 경험이 있었으며, 일부 응답자는 혐오표현을 유행어로 인식해 직접 사용한 경험도 있다고 답했다. 이는 혐오표현에 대한 사회 전반의 감수성이 둔화되고 있음을 시사한다. (김 et al. 501)

더욱이 최근의 혐오표현은 노골적인 욕설에 그치지 않는다. 밈이나 가벼운 유머의 형태로 일상에 스며들며, 수용자의 문제 인식 자체를 무력화한다. 관련 연구에 따르면 혐오가 유머로 포장될 경우 수용자는 이를 문제로 인식하지 못하고 침묵하는 경향이 강화되며, 이는 혐오 대상에 대한 부정적 감정을 자극하고 편견을 조장하는 방향으로 작용할 수 있다. (연 and 이 219) 즉, 혐오표현의 위험성은 표현의 과격함이 아니라 일상 언어 속에 자연스럽게 녹아들어 문제 인식 자체를 차단한다는 데 있다.

이러한 현실에서 짧은 시간에 수백만 건의 게시물이 생성되는 소셜미디어 환경을 인력으로 관리하기에는 한계가 뚜렷하다. 단순 키워드 필터링은 문장의 맥락을 반영하지 못해 오분류 가능성이 크고, BERT와 같은 사전학습 언어 모델을 활용한 자동 탐지 접근이 높은 분류 정확도를 보여 주목받고 있다 (Devlin et al., 2019; Shawkat et al., 2024). 그러나 이들 모델에도 근본적 한계가 존재한다. 첫째, 혐오(hate speech)와 단순 공격적 언어(offensive language)를 구분하는 문제에서 여전히 높은 오분류율을 보인다 (Davidson et al., 2017). Davidson et al.의 연구에서 전체 F1은 0.90이었으나 hate class의 F1은 0.44에 불과했으며, Mathew et al. (2021)의 HateXplain 벤치마크에서도 BERT는 분류 성능은 양호하나 모델이 주목하는 토큰이 인간 판단 근거와 상당히 괴리되는 현상이 관찰되었다. 둘째, 기존 모델은 '혐오 여부'만을 출력할 뿐, 왜 해당 텍스트를 혐오로 판단했는지에 대한 근거를 제시하지 못한다. 판단 근거가 부재한 탐지 결과는 플랫폼 운영자의 검수 효율을 저하시키고, 과잉 차단이라는 반발을 초래하며, 유머로 포장된 암묵적 혐오의 경우 왜 문제인지를 수용자 스스로 인식하지 못하는 상황을 방치한다.

따라서 혐오표현의 자동 탐지에는 높은 분류 정확도와 함께, 모델의 판단 과정을 투명하게 설명하고, 그 설명을 통해 모델 자체를 체계적으로 개선하는 접근이 동시에 필요하다.

### 1.2 프로젝트 목표

본 프로젝트는 혐오표현 탐지 모델의 오분류 원인을 설명 가능한 AI(Explainable AI, XAI) 기법으로 진단하고, 그 진단 결과에 기반하여 모델을 개선한 뒤, 개선 효과를 다시 XAI로 검증하는 순환적 프레임워크를 설계/구현하는 것을 목적으로 한다.

구체적으로, 본 프로젝트는 다음의 세 단계로 구성된다.

첫째, 진단 단계에서는 BERT 기반 분류 모델에 SHAP(SHapley Additive exPlanations)과 LIME(Local Interpretable Model-agnostic Explanations)을 적용하여, 모델이 혐오(hate)와 공격적 언어(offensive)를 오분류하는 사례에서 어떤 텍스트 요소에 과도하게 의존하는지를 식별한다. 두 기법의 상위 중요 토큰 간 일치도(Overlap@5)를 교차 검증함으로써 진단의 신뢰성을 확보한다.

둘째, 개선 단계에서는 진단에서 도출된 취약점, 즉 모델이 텍스트의 감성적 맥락을 충분히 반영하지 못한다는 문제를 해결하기 위해, 외부 감성 분석 도구(VADER)의 감성 강도 피처를 BERT 임베딩에 결합한 개선 모델을 구축한다. 이 접근은 추가적인 대규모 사전학습 없이도 경량 피처 보강만으로 hate/offensive 구분 성능을 향상시키는 것을 목표로 한다.

셋째, 검증 단계에서는 개선 전후 모델에 동일한 XAI 분석을 재적용하여, 추가된 감성 피처가 실제로 분류 판단에 기여하는지를 정량적으로 비교한다. 이를 통해 단순한 성능 수치 향상을 넘어, 개선의 메커니즘 자체를 설명 가능한 형태로 제시한다.

분석 대상으로는 HateXplain 데이터셋(Mathew et al., 2021)을 사용한다. 이 데이터셋은 혐오(hate), 공격적 언어(offensive), 정상(normal)의 3-class 분류 체계와 함께 인간 주석자의 판단 근거(rationale)를 포함하고 있어, XAI 기반 분석의 ground truth로 활용할 수 있다. 다수결 라벨링 후 undecided를 제외하면 약 13,433건을 70/10/20 비율로 stratified split하여 사용한다. 분석 언어는 영어로, 공개 데이터셋과 선행 연구의 축적이 풍부하여 방법론의 유효성을 객관적으로 검증하기에 적합하기 때문이다.

본 프로젝트를 통해 혐오표현 탐지의 정확도 향상과 판단 투명성 확보를 동시에 달성하고, 나아가 XAI를 사후 해석 도구가 아닌 모델 개선의 능동적 수단으로 활용하는 방법론적 기반을 마련하고자 한다.

# 버전 B: 실무 프로젝트 톤 (보다 직관적이고 간결한 문체)

## 1. 프로젝트 개요

### 1.1 수행 배경 및 필요성

소셜미디어가 일상적 소통 공간으로 자리 잡으면서, 특정 집단을 겨냥한 혐오표현도 함께 확산되고 있다. 국내에서는 2010년 전후부터 반다문화 인터넷 커뮤니티를 중심으로 외국인과 이주노동자를 향한 적대적 언어가 사회적 갈등 요인으로 부상했고, 이후 그 대상은 성별, 세대, 지역 등으로 확대되었다. 관련 조사에서는 참여 대학생 전원이 온라인에서 혐오표현을 접한 경험이 있었으며, 일부는 이를 유행어처럼 인식해 직접 사용한 적도 있다고 응답했다. (김 et al. 501) 이는 혐오표현에 대한 사회적 감수성이 뚜렷이 둔화되고 있음을 보여준다.

특히 우려스러운 점은 최근의 혐오표현이 직접적 욕설보다 밈이나 유머의 형태로 일상에 스며들고 있다는 것이다. 연구에 따르면 유머로 포장된 혐오는 수용자의 문제 인식을 무력화하고, 혐오 대상에 대한 부정적 감정과 편견을 오히려 강화하는 방향으로 작용한다. (연 and 이 219) 혐오표현의 위험성은 과격함에 있는 것이 아니라, 그것이 일상 언어 속에 녹아들어 인식의 경계 자체를 허문다는 데 있다.

이에 대응하여 BERT 등 사전학습 언어 모델 기반의 자동 탐지 기술이 발전해왔으나, 두 가지 구조적 한계가 남아 있다.

한 가지는 정확도의 한계이다. Davidson et al. (2017)의 연구에서 전체 F1은 0.90이었으나, 실제 혐오 발화(hate speech)만 놓고 보면 F1이 0.44로 급락했다. 혐오와 단순 공격적 표현(offensive language)의 경계가 모호한 사례에서 모델이 체계적으로 실패하는 것이다. 예를 들어, 특정 슬랭이 맥락에 따라 혐오일 수도 있고 일상적 표현일 수도 있는 경우, 현재의 모델은 텍스트의 감성적 뉘앙스를 충분히 포착하지 못한다.

다른 한 가지는 설명의 부재이다. 기존 모델은 '혐오 여부'만 출력할 뿐, 왜 해당 텍스트가 혐오인지에 대한 판단 근거를 제시하지 못한다. 이는 플랫폼 운영자의 의사결정을 어렵게 하고, 사용자로부터 과잉 차단이라는 반발을 초래하며, 무엇보다 유머로 포장된 암묵적 혐오가 왜 문제인지를 수용자에게 전달할 수단이 없다는 것을 의미한다.

결국, 혐오표현 탐지에서 필요한 것은 높은 분류 성능만이 아니다. 모델이 왜 틀렸는지를 진단하고, 그 진단을 기반으로 모델을 고치고, 고친 결과가 왜 나아졌는지를 다시 설명할 수 있는 체계적 접근이 필요하다.

### 1.2 프로젝트 목표

본 프로젝트의 목표는 혐오표현 탐지에서 "분류 -> 진단 -> 개선 -> 검증"의 순환 구조를 갖춘 XAI 기반 파이프라인을 설계하고 구현하는 것이다.

기존 연구들은 모델의 분류 성능을 높이는 데 집중하거나(Shawkat et al., 2024; Caselli et al., 2021), 학습된 모델에 XAI를 사후적으로 적용해 해석하는 데 그쳤다(Mathew et al., 2021). 그러나 "XAI가 보여준 문제를 실제로 어떻게 고칠 것인가", 그리고 "고친 뒤 XAI로 다시 보면 무엇이 달라졌는가"라는 질문에 답한 연구는 드물다. 본 프로젝트는 이 빈 공간을 채우고자 한다.

이를 위해 다음 세 단계의 파이프라인을 설계한다.

**진단 단계**에서는 BERT를 HateXplain 데이터셋(Mathew et al., 2021)의 hate/offensive/normal 3-class 분류 과제에 미세조정(fine-tuning)한 뒤, SHAP과 LIME 두 가지 XAI 기법을 동시에 적용한다. 오분류된 샘플에서 두 기법이 공통으로 지목하는 상위 중요 토큰을 비교(Overlap@5)하여, 모델이 어떤 패턴에 과의존하고 있는지를 신뢰성 있게 진단한다.

**개선 단계**에서는 진단 결과를 바탕으로, 모델이 놓치고 있는 정보가 무엇인지를 특정하고 이를 보완하는 피처를 설계한다. 구체적으로, 감성 분석 도구 VADER(Hutto and Gilbert, 2014)의 4차원 감성 강도 점수(pos, neg, neu, compound)를 BERT 임베딩에 결합한 `HybridSentimentClassifier`를 구축하여, hate와 offensive 간의 감성적 뉘앙스 차이를 모델이 포착할 수 있도록 한다.

**검증 단계**에서는 개선 전후 모델에 동일한 SHAP/LIME 분석을 재적용하여, 새로 추가된 감성 피처의 기여도를 정량적으로 측정하고, 진단 단계에서 발견된 오분류 패턴이 실제로 완화되었는지를 확인한다. 이를 통해 성능 향상의 원인을 "감성 피처가 이만큼 기여했다"는 수치로 제시하는 것이 핵심이다.

분석 데이터로는 Twitter와 Gab에서 수집된 약 2만 건의 영어 텍스트로 구성된 HateXplain 벤치마크를 사용한다. 다수결 라벨링 후 undecided를 제외하면 약 13,433건이 되며, 이를 **70/10/20 비율**로 stratified split한다. 이 데이터셋은 3-class 분류 라벨, 대상 커뮤니티 정보, 인간 판단 근거(rationale)를 모두 포함하여 XAI 분석의 정량적 검증에 적합하다.

본 프로젝트를 통해 XAI를 단순한 사후 해석이 아닌, 모델 개선을 이끄는 능동적 진단 도구로 활용하는 방법론을 제시하고, 혐오표현 탐지 결과에 대한 투명한 근거를 함께 제공함으로써 안전한 디지털 소통 환경 조성에 기여하고자 한다.

## 2. 시스템 아키텍처 및 파일 구조

### 2.1 코드베이스 구조

리팩토링된 코드베이스는 4개의 핵심 Python 모듈로 구성되며, `run.sh`를 통해 단계별로 실행한다.

```
HateSpeachStudy/
├── run.sh                    # CLI 실행기 (모든 단계의 진입점)
├── run_experiments.py        # Python 엔트리포인트
├── experiment_core.py        # 데이터, 모델, 학습, 벤치마크
├── experiment_xai.py         # SHAP + LIME + Overlap@5
├── experiment_dashboard.py   # HTML 대시보드 생성
├── utils.py                  # 공통 유틸리티
│
├── compat/                   # 이전 s00~s09, stage01~04 래퍼 (호환용)
├── data/                     # HateXplain 원본 (자동 다운로드)
├── checkpoints/              # 모델 체크포인트 (.pt)
└── outputs/
    ├── reports/              # 벤치마크 요약, 데이터 프로파일
    ├── tuning/               # 하이퍼파라미터 탐색 결과
    ├── runs/                 # seed별 반복 실험 결과
    ├── xai/                  # SHAP/LIME 분석 결과, Overlap@5
    ├── dashboard/            # HTML 대시보드 (index.html)
    └── logs/                 # 실행 로그
```

> 이전 `s00_data.py ~ s09_xai_phase2.py`, `stage01~04_*.py` 스크립트는 `compat/` 디렉토리로 이동되었으며, 호환 래퍼로만 기능합니다. 실제 로직은 모두 4개의 핵심 모듈에 통합되어 있습니다.

### 2.2 파이프라인 실행 흐름

```
data → vader → [tune] → benchmark → freeze-study → xai → dashboard
```

| 명령어 | 하는 일 |
|--------|---------|
| `./run.sh data` | HateXplain 다운로드, 다수결 라벨링, 70/10/20 stratified split |
| `./run.sh vader` | VADER 감성 피처 4차원(pos, neg, neu, compound) 추출 |

### 2.3 모델 목록

| 모델 | 클래스 | 아키텍처 |

### 2.4 XAI 접근

- **SHAP** (Lundberg & Lee, 2017): 각 토큰의 예측 기여도를 Shapley value로 산출
- **LIME** (Ribeiro et al., 2016): 개별 예측 주변에서 해석 가능한 선형 모델로 근사
- **Overlap@5**: SHAP Top-5 토큰과 LIME Top-5 토큰의 일치도를 비교하여 교차 검증
- Before/After 비교를 통해 VADER 피처의 실제 기여도를 정량적으로 측정

# 표절 위험 요소 제거 확인

두 버전 모두 다음 사항을 반영하여 작성했습니다:

1. **Cheng (2022) 차별화:** "감성 피처 결합 자체"가 아닌 "XAI 진단 기반의 순환적 프레임워크"를 핵심 목표로 설정
2. **HateXplain 원논문 차별화:** 원논문의 LIME 분석은 사후 해석에 그쳤으나, 우리는 진단->개선->재진단의 순환 구조를 명시
3. **HateXplain 2.0 차별화:** 아키텍처 변경이 아닌 외부 피처 보강 + post-hoc XAI 기반 접근
4. **SFA (2023) 차별화:** SHAP values 자체를 피처로 사용하지 않고, 진단 결과의 해석을 통해 외부 피처를 선택하는 인간-in-the-loop 접근
5. **독창성 강조 포인트:** Before/After SHAP 비교를 통한 개선 메커니즘의 정량적 검증
