```
./run.sh [command]

            ├── experiment_core.py    — 데이터, 모델, 학습, 벤치마크, 튜닝, freeze study

## Stage별 상세 흐름

### Stage 1: `data` — 데이터 전처리

```
data/dataset.json (HateXplain 원본 20,148건)
    │
    ├── _majority_label()     3인 annotator 다수결 투표
    ├── undecided 제외         → 19,192건
    ├── @멘션 익명화            → <user> 치환
    ├── 중복 텍스트 제거
    │
    └── prepare_data()
        ├── stratified split: 70/10/20 (seed=42)
        │   ├── train: 13,433건
        │   ├── val:    1,920건
        │   └── test:   3,839건
        │
        ├── outputs/data_splits.pkl          (분할 데이터 캐시)
        ├── outputs/reports/data_profile.json (통계 요약)
        └── outputs/reports/eda/split_distribution.png
```

**DataFrame 컬럼:** `post_id`, `text`, `label` (0/1/2), `label_name`, `targets`

**라벨 매핑:** 0=hatespeech, 1=offensive, 2=normal

**분포:** hatespeech 30.9% / offensive 28.5% / normal 40.6%


### Stage 2: `vader` — VADER 감성 피처 추출

```
train/val/test 텍스트
    │
    └── extract_vader_features()
        ├── NLTK SentimentIntensityAnalyzer
        ├── 각 텍스트 → [neg, neu, pos, compound] 4차원 벡터
        │
        └── outputs/vader_features.pkl
            └── {"train": (13433, 4), "val": (1920, 4), "test": (3839, 4)}
```

**용도:** BERT+VADER, RoBERTa+VADER 하이브리드 모델의 추가 입력 피처


### Stage 3: `eda` — 탐색적 데이터 분석

```
experiment_eda.run_eda()
    ├── _analyze_text_length()          텍스트 길이 분포 (클래스별)
    ├── _analyze_vader_by_class()       VADER 감성 점수 분포 (클래스별)
    ├── _analyze_targets()              대상 커뮤니티 분석
    ├── _analyze_vocabulary_overlap()   클래스 간 어휘 겹침 분석
    ├── _analyze_class_distribution()   라벨 분포 시각화
    ├── _analyze_ngrams()               n-gram 빈도 분석
    ├── _analyze_wordcloud()            워드클라우드 생성
    ├── _analyze_rationale_distribution() Human rationale 분포
    ├── _analyze_vader_separability()   VADER 클래스 분리력 분석
    └── _generate_summary()             EDA 요약 JSON 생성
```

**출력:** `outputs/reports/eda/` 하위 PNG + JSON 파일들


### Stage 4: `tune` — 하이퍼파라미터 탐색

```
run_hyperparameter_tuning()
    │
    ├── 대상 모델: BERT-base, BERT+VADER, RoBERTa+VADER
    ├── 탐색 축 (sequential):
    │   ├── learning_rate: [1e-5, 2e-5, 3e-5]
    │   ├── dropout: [0.1, 0.2, 0.3]
    │   └── (batch_size=64, epochs=5 고정)
    │
    ├── 각 후보값마다 seed=42로 1회 학습 → val macro_f1 기준 선택
    │
    └── outputs/tuning/
        ├── transformer_tuning_log.csv       전체 탐색 로그
        ├── transformer_tuning_best.json     모델별 최적 파라미터
        └── {model}/{param}/metrics.json     개별 실험 결과
```

**현재 최적값:**

| 모델 | lr | dropout | batch | epochs |
|---|---|---|---|---|
| BERT-base | 3e-5 | 0.1 | 64 | 5 |
| BERT+VADER | 2e-5 | 0.1 | 64 | 5 |
| RoBERTa+VADER | 2e-5 | 0.1 | 64 | 5 |


### Stage 5: `benchmark` — 반복 벤치마크

```
run_benchmark()
    │
    ├── TF-IDF 베이스라인 (sklearn):
    │   ├── TF-IDF + LR     (3 seeds × 결정적 = 동일 결과)
    │   └── TF-IDF + SVM    (3 seeds × 결정적 = 동일 결과)
    │
    ├── Transformer 모델 (PyTorch, MPS):
    │   ├── BERT-base             (TransformerCLSClassifier)
    │   ├── BERT+MLP              (TransformerMLPClassifier, ablation 통제)
    │   ├── BERT+VADER            (HybridSentimentClassifier)
    │   └── RoBERTa+VADER         (HybridSentimentClassifier)
    │
    ├── 각 모델 × 3 seeds (42, 52, 62):
    │   ├── train_neural_model()
    │   │   ├── CrossEntropyLoss(weight=class_weights, label_smoothing=0.1)
    │   │   ├── AdamW + linear warmup scheduler
    │   │   ├── EarlyStopping(patience=2)
    │   │   └── best checkpoint 저장 (val macro_f1 기준)
    │   │
    │   └── evaluate_neural_model() → test set 평가
    │       ├── macro_f1, accuracy, auroc, per_class 메트릭
    │       └── confusion matrix + learning curve 저장
    │
    ├── 집계:
    │   ├── 모델별 mean ± std
    │   ├── paired t-test (모든 조합, alpha=0.05)
    │   └── best model 선정 (macro_f1 기준)
    │
    └── outputs/reports/
        ├── benchmark_runs.csv          18행 (6모델 × 3seeds)
        ├── benchmark_summary.csv       6행 (모델별 요약)
        ├── benchmark_summary.md        마크다운 표
        ├── best_models.json            각 모델의 best seed 정보
        ├── significance_tests.csv      15쌍 t-test 결과
        ├── model_comparison.png        Macro F1 바 차트
        └── per_class_f1_heatmap.png    클래스별 F1 히트맵
```

**Class Weighting 메커니즘:**
```
compute_class_weight_tensor(labels, imbalance_threshold=0.40)
    ├── minority_ratio = min_class / total < 0.40 → True
    ├── balanced weights: hate=1.08, offensive=1.17, normal=0.82
    └── CrossEntropyLoss(weight=tensor, label_smoothing=0.1)
```

### Stage 7: `xai` — 설명가능 AI 분석

```
run_xai()
    │
    ├── Step 1: 테스트셋 전체 예측
    │   ├── Baseline: BERT-base (best seed 체크포인트)

    │
    ├── Step 2: 분석 대상 샘플 선정 (50개)
    │   ├── 우선순위: fixed_error (Baseline 오답 → Improved 정답)

    │   ├── Improved SHAP
    │   └── Improved LIME
    │

## 모델 아키텍처 요약

| 모델 | 클래스 | 인코더 | 추가 입력 | 분류기 |
|---|---|---|---|---|
| TF-IDF+LR | sklearn | TF-IDF (10K) | — | LogisticRegression |
| TF-IDF+SVM | sklearn | TF-IDF (10K) | — | LinearSVC |
| BERT-base | TransformerCLSClassifier | bert-base-uncased | — | Linear(768→3) |
| BERT+MLP | TransformerMLPClassifier | bert-base-uncased | — | MLP(768→256→3) |
| BERT+VADER | HybridSentimentClassifier | bert-base-uncased | VADER 4D | MLP(768+4→256→3) |
| RoBERTa+VADER | HybridSentimentClassifier | roberta-base | VADER 4D | MLP(768+4→256→3) |

**BERT+MLP의 역할:** VADER 없이 동일 MLP 구조 → "VADER 추가 효과"를 분리하기 위한 ablation 통제 모델

## XAI 이중 검증 체계 (현재 구현 상태)

```
                    ┌─────────────────────────────────┐
                    │      XAI 이중 검증 체계          │
                    └─────────────────┬───────────────┘
                                      │
              ┌───────────────────────┼───────────────────────┐
              │                       │                       │
     ┌────────┴────────┐    ┌────────┴────────┐    ┌────────┴────────┐
     │  설명 안정성      │    │  설명 타당성      │    │  오분류 분석     │
     │  (Stability)     │    │  (Validity)      │    │  (Error Fix)    │
     └────────┬────────┘    └────────┬────────┘    └────────┬────────┘
              │                       │                       │
     SHAP Top-5 ∩ LIME Top-5  Model Top-5 vs Human    Baseline 오답 →
     Jaccard >= 0.6            Rationale >= 0.5        Improved 정답
              │                       │                       │
     Baseline: 0.628           SHAP:  0.579 → 0.688    25/50 건
     Improved: 0.724           LIME:  0.725 → 0.692    교정됨
```

**핵심 발견:**
- RoBERTa+VADER는 SHAP 기준 Human Rationale 정렬이 0.579→0.688로 대폭 개선
- 이는 VADER 감성 피처가 모델을 "인간이 보는 근거"에 더 가깝게 유도한다는 증거
- LIME 기준에서는 소폭 하락(0.725→0.692)하지만, SHAP과 LIME 간 격차가 줄어들어 설명 안정성이 동시에 향상
