# 예상 질문과 답변 (FAQ)

> 이 프로젝트를 평가하거나 발표에서 받을 법한 질문을 미리 정리했습니다.
> 핵심 태도는 **과장하지 않는 것** — "작지만 통제된 반복 실험에서 통계적으로 확인된 개선"으로 서술합니다.

## 한 문장 결론

HateXplain 3분류 혐오표현 탐지에서, 기준 BERT 대비 **근거 주석 기반 어텐션 손실**이 작지만 **15시드 반복 기준으로 안정적이고 통계적으로 유의한** 성능 개선(`A_B` 0.6798 → `B_B` 0.6858)을 보였습니다.

## 발표자가 외울 숫자

| 항목 | 값 |
|---|---|
| 데이터 | HateXplain 19,192건, 3-class, split seed 42로 70/10/20 |
| 실험량 | 8조건 × 15 training seed = 120 runs |
| 최고 모델 | `B_B` = BERT + 근거 정렬 어텐션 손실 |
| Macro F1 | `A_B` 0.6798 → `B_B` 0.6858 (**+0.0060**) |
| 통계 | `A_B` vs `B_B` Holm 보정 p = **0.0027**, Cohen's d = 1.35 |
| ANOVA | 백본·어텐션 손실 유의, VADER 유의하지 않음 |

---

## 성능과 통계

**Q. 개선폭이 너무 작은 것 아닌가?**
맞습니다. 그래서 "큰 성능 향상"이 아니라 **"작지만 일관된 개선"**이라고 말합니다. 이 연구의 가치는 과장된 SOTA가 아니라 **반복 실험 + 통계 검정 + 설명가능성 검증 체계**에 있습니다.

**Q. 통계적으로 유의하다는 근거는?**
동일 split 위에서 15개 training seed를 반복하고, `A_B` vs `B_B`를 **동일 시드 paired 비교**했습니다. Holm 보정 후 p = 0.0027로 유의했습니다.

**Q. Holm 보정은 왜 썼나?**
여러 조건을 동시에 비교하면 우연히 p < 0.05가 나올 확률이 커집니다. Holm은 다중 비교에서 false positive를 줄이는 보정입니다. 핵심 결론의 주연이 아니라 과대해석을 막는 보조 확인값으로 둡니다.

**Q. Effect size가 1.35인데 개선폭은 0.006이면 모순 아닌가?**
아닙니다. effect size(Cohen's d)는 *평균 차이 ÷ 시드 간 차이의 표준편차*입니다. 절대 차이가 작아도 **시드별 차이의 변동이 매우 작으면** effect size는 크게 나올 수 있습니다. 즉 "작지만 매우 일관된 개선"이라는 뜻입니다.

**Q. RoBERTa가 BERT보다 낮은 이유는?**
데이터 규모·하이퍼파라미터, rationale mask의 서브워드 매핑 차이, base model의 도메인 적합성 차이 가능성이 있습니다. 3요인 ANOVA에서도 **백본이 가장 큰 요인**(η² ≈ 0.39)이었습니다.

**Q. 왜 split seed는 하나만 썼나?**
조건 간 비교를 공정하게 하려면 train/val/test를 고정해야 합니다. 대신 **training seed 15개**를 바꿔 모델 초기화·학습 랜덤성에 대한 안정성을 확인했습니다. split seed를 여러 개로 바꾸는 nested 반복은 향후 과제로 남겨둔 한계입니다.

## VADER

**Q. VADER는 왜 효과가 없었나?**
혐오표현은 단순 부정 감성이 아니라 **집단 표적·맥락·의도**의 문제입니다. VADER는 sentiment 신호라 hate/offensive 구분에 충분하지 않았습니다. 3요인 ANOVA에서 VADER 단독 효과는 유의하지 않았습니다(p ≈ 0.52). **이 "약함" 자체가 혐오표현 탐지에서 단순 감성정보의 한계를 보여주는 결과**입니다.

**Q. 그럼 VADER는 왜 선택했나? XAI 진단 결과인가?**
아닙니다. VADER는 Cheng(2022) 선행연구 기반 **사전 가설**로 선택했고, ablation으로 그 가설을 통제 검증한 것입니다. XAI는 설계에 되먹이지 않고 사후 검증으로만 썼습니다.

## XAI

**Q. 최종 성능 모델은 `B_B`인데 XAI는 왜 `A_B` vs `D_B` 중심인가?**
XAI 비교축은 baseline 대비 **두 처방을 모두 적용한** full-intervention 조건(`D_B`)으로 사전 설계했습니다. 최종 성능 결론(`B_B`)과 XAI 비교축(`A_B` vs `D_B`)은 **분리해서** 설명합니다.

**Q. 그럼 XAI 결론은 약한 것 아닌가?**
XAI는 seed·sample 수가 제한적이라 정량 일반화 근거가 아니라 **qualitative/exploratory evidence**입니다. 성능 결론의 주근거는 120-run 벤치마크와 통계 검정이고, XAI는 판단 경향과 human rationale 정렬 여부를 설명하는 보조 근거입니다.

**Q. rationale을 학습에 쓰면 test leakage 아닌가?**
아닙니다. rationale은 **train에서만 보조 supervision**으로 쓰고, 추론 시에는 텍스트만 입력합니다. test label/rationale을 학습에 쓰면 leakage지만, 설계상 그렇게 하지 않았습니다.

**Q. 어텐션이 human rationale에 맞춰졌다고 모델이 진짜 맥락을 이해한 건가?**
완전히 그렇다고 말할 수 없습니다. 다만 **단어 하나에만 의존하는 패턴을 줄이고**, human rationale과 일부 정렬되는 방향의 학습 신호를 줬다는 수준으로 서술합니다.

## 포지셔닝

**Q. SOTA와 비교하면 어디쯤인가?**
외부 SOTA 달성 주장은 하지 않습니다. binary toxic/non-toxic F1 0.81과는 task·metric이 다르며, 본 연구는 **3-class Macro F1 기준 내부 실험군 비교**입니다.

**Q. 향후 개선 방향은?**
HateBERT/DeBERTa 계열 백본, class-weight/focal loss, rationale auxiliary head, 더 큰 XAI sample·seed, split seed 반복, binary/3-class 병렬 평가.

## 절대 피하는 표현 vs 권장 표현

| 피하는 표현 | 권장 표현 |
|---|---|
| "SOTA 달성했다" | "실험군 내 최고 성능은 `B_B`, baseline 대비 작지만 유의한 개선" |
| "성능이 크게 개선되었다" | "개선폭은 제한적이라 실무 적용 전 추가 개선 필요" |
| "XAI가 성능 향상을 입증했다" | "XAI는 성능 근거가 아니라 판단 경향을 설명하는 보조 evidence" |
| "`D_B`가 최종 모델이다" | "최종 성능 모델은 `B_B`, XAI 비교축은 `A_B` vs `D_B`" |
| "VADER가 효과적이었다" | "VADER 효과가 약했다는 점이 단순 감성정보의 한계를 보여준다" |

## 용어 부록

| 용어 | 한 줄 설명 |
|---|---|
| **SHAP** | 각 토큰이 예측을 얼마나 밀어올렸는지 기여도를 계산 |
| **LIME** | 문장 일부를 바꾼 주변 샘플로 복잡한 모델을 선형 설명으로 근사 (SHAP과 교차 확인) |
| **LOO** | Leave-One-Out. 토큰 하나를 빼고 예측이 얼마나 흔들리는지 → 단어 의존도 |
| **Comprehensiveness** | 중요 토큰을 제거했을 때 예측 확신이 떨어지는 정도 (faithfulness) |
| **Rationale overlap** | 사람이 근거로 표시한 토큰과 모델이 중요하게 본 토큰의 겹침 |
| **CI (Concentration Index)** | 중요도가 소수 토큰에 몰려 있는 정도 → 높으면 단어 과의존 |
| **Seed stability** | 시드가 바뀌어도 설명이 유지되는지 |
| **Holm 보정** | 다중 비교에서 우연한 유의 결과를 줄이는 p-value 보정 |
| **ANOVA** | 여러 설계 요인 중 무엇이 성능 차이를 설명하는지 보는 분산분석 |
| **Effect size (Cohen's d)** | 평균 차이를 표준편차로 나눈 값 — 절대 차이가 작아도 변동이 작으면 크게 나옴 |
